# 📚 Obsidian Asset Collector & Merger

> **Obsidian** 사용자를 위한 재귀적 문서 수집 및 NotebookLM 최적화 병합 도구

이 도구는 옵시디언 볼트 내의 마크다운 문서를 분석하여, 연결된 모든 **리소스(이미지 등)를 수집**하고 여러 문서를 **하나의 마크다운 파일로 병합**해줍니다. 특히 **NotebookLM**의 50개 파일 제한을 우회하여 방대한 지식을 학습시키고자 할 때 유용합니다.

---

## ⚠️ 호환성 안내 (Compatibility)

- **운영체제**: 본 도구는 **Windows 10/11** 환경에 최적화되어 있습니다.
    
- **파일 형식**: 마크다운(`.md`) 문서 및 일반적인 이미지 리소스 파일들을 지원합니다.
    

---

## 🌟 주요 기능

- **재귀적 탐색 (Recursive Search)**: 선택한 메인 문서뿐만 아니라 그 문서가 링크하고 있는 하위 문서들을 끝까지 추적하여 분석합니다.
    
- **스마트 리소스 수집**: 문서에 포함된 리소스(이미지 등)를 `exported_assets` 폴더로 모아줍니다. 이미 수집된 파일은 자동으로 스킵하여 작업 속도를 극대화합니다.
    
- **NotebookLM 최적화 병합**: 모든 마크다운 내용을 `merged_notebook.md`라는 단일 파일로 합쳐줍니다. 문서별로 구분 헤더를 삽입하여 AI가 출처를 명확히 인식하게 돕습니다.
    
- **전용 볼트 브라우저**: 볼트 경로를 벗어나지 않는 안전하고 직관적인 전용 파일 탐색기 UI를 제공합니다.
    

---

## 🚀 시작하기 전에 (Python 설치)

이 도구를 실행하려면 **Python 3**가 설치되어 있어야 합니다.

1. [Python 공식 홈페이지](https://www.python.org/downloads/)에서 설치 파일을 다운로드합니다.
    
2. 설치 시 **"Add Python to PATH"** 체크박스를 반드시 선택하세요.
    
3. 터미널(CMD)에서 `python --version`을 입력하여 설치를 확인합니다.
    

---

## 🛠 사용 방법

### 1. 프로젝트 구조 확인

아래와 같이 파일이 배치되어 있는지 확인하세요.

Plaintext

```
Project_Root/
├── run_collector.bat       # 실행용 윈도우 배치 파일
└── origin/
    └── collect_markdown_assets.py  # 메인 로직 스크립트
```

### 2. 실행 순서

1. 결과물을 저장하고 싶은 폴더(CMD 환경)에서 `run_collector.bat`를 실행합니다.
    
2. **볼트 선택**: 본인의 옵시디언 볼트 최상위 폴더를 선택합니다.
    
3. **시작 문서 선택**: 전용 브라우저 창에서 분석을 시작할 **메인 마크다운 파일**을 선택합니다.
    
4. **결과 확인**: 실행한 위치에 다음 결과물이 생성됩니다.
    
    - `exported_assets/`: 수집된 이미지 및 리소스 파일들
        
    - `merged_notebook.md`: NotebookLM 업로드용 통합 문서
        

---

## 💡 활용 팁

- **NotebookLM 업로드**: 생성된 `merged_notebook.md` 파일 하나만 NotebookLM 소스로 등록하세요. 파일 개수 제한(50개) 없이 수백 개의 문서 내용을 한 번에 학습시킬 수 있습니다.
    
- **비-윈도우 환경**: macOS나 Linux 사용자는 터미널에서 아래 명령어로 직접 실행 가능합니다.
    
    - `python origin/collect_markdown_assets.py "$(pwd)"`
        
    - _(단, `tkinter` 패키지 설치가 필요할 수 있습니다.)_
